Design Choices

I collected location data, battery data, movements data, app usage data and wifi usage data using the aware framework. I picked the location data, app usage and movements to do my visualization because they represent three orthogonal aspects of how I was spending the week. I chose Tableau's map visualization to visualize my location since the map gives a context of where I had been. I use bubble charts to visualize app usage where the size of the bubble represents the proportion of time I spent on an app because it is intuitive and visually appealing. I use a line plot to visualize the number of significant movements every hour because the data is a time series and a line plot reflects the fluctuation with respect to time.

Data Quality

The data quality is accessed and ensured via EDA using pandas and Tableau visualization. The following section describes the 4 C's in detail.

Completeness The data collected with the default sensors is somewhat sufficient to answer the question. However, I was not able to install additional plugins like Google's activity recognition which could provide more insights into answering the question. I carefully cleaned the data so that they serve a better purpose at answering my question. For example, in the Tableau App usage visualization, I excluded Android system apps from the plot. I did not notice any junk values during the EDA and visualizing the data with Tableau. There might be missing values due to my phone being off or Aware not able to sync the data with MySQL database. The plot shows the number of records found in MySQL databases versus day. that more data was collected through October 8th to 10th than the rest of the week.

Coherence Since the data Aware collected has a schema, there is little room for misbehaved data. Also, a lot of data, like battery status, application usage comes from the Android system, I assume the Android system is robust enough to deliver coherent data to the Aware client. I also made sure that the data matches my recollection of what happened last week through Tableau visualization. I did need to convert the Unix timestamp Aware collected to date time using pandas.to_datatime() function to make the timestamp interpretable.

Correctness The bias largely comes from the data collection process. The mobile data I was able to collect is limited by the sensors I have on my phone and their accuracy. Also, I am making an assumption that my phone is always charged, Aware can always sync the sensor data with the MySQL backend, and I was carrying my phone all the time. These assumptions are only partially true since the plot above shows that the number of sensor records found in MySQL database varies from day to day, and I generally leave my phone charging while I am at home. Though it is difficult to fix the data collection bias in hindsight, I carefully picked the data I visualized so that the represents three orthogonal aspects of my behavior during the last week.

Accountability The data was collected by the aware framework and stored in MySQL backend. I have access to both the original data stored in MySQL and cleaned data stored on my localhost. While I am the only person having accessibility to the original data, the Tableau visualizations are accessible to the public.
